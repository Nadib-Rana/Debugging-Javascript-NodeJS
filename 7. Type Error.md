### টাইপ এরর: এটি কী এবং কীভাবে ঠিক করবেন

#### টাইপ এরর কী?
**টাইপ এরর** ঘটে যখন আপনি এমন একটি মানের উপর অপারেশন করার চেষ্টা করেন যা সেই অপারেশনের জন্য সঠিক ডেটা টাইপ নয়। এটি সাধারণত ডেটা টাইপের ভুল ব্যবহারের কারণে ঘটে।

#### টাইপ এররের সাধারণ কারণ:
1. **ভুল ডেটা টাইপের উপর অপারেশন করা**: যেমন, একটি নম্বরের উপর স্ট্রিং-এর মেথড ব্যবহার করা।
2. **অস্তিত্বহীন মেথড কল করা**: সেই ডেটা টাইপের জন্য মেথডটি প্রযোজ্য নয়।
3. **ভুল ডেটা স্ট্রাকচারের ব্যবহার**: যেমন, অ্যারে বা অবজেক্টকে সংখ্যার মতো ব্যবহার করা।

#### টাইপ এররের উদাহরণ:

```javascript
let num = 10;
num.toUpperCase();  // টাইপ এরর: 'toUpperCase()' স্ট্রিং মেথড, কিন্তু 'num' একটি সংখ্যা
```

#### টাইপ এরর কীভাবে ঠিক করবেন:
1. **সঠিক ডেটা টাইপ ব্যবহার করুন**: নিশ্চিত করুন যে আপনি সঠিক টাইপের ডেটার উপর অপারেশন করছেন।
2. **মেথডের প্রযোজ্যতা পরীক্ষা করুন**: যে মেথডটি কল করছেন তা সেই টাইপের জন্য প্রযোজ্য কিনা তা যাচাই করুন।
3. **ডিবাগ করুন**: `console.log()` ব্যবহার করে ডেটা টাইপ পরীক্ষা করুন অপারেশন করার আগে।

#### সঠিক কোড:

```javascript
let text = "Hello";
text.toUpperCase();  // এটি কাজ করবে কারণ 'text' একটি স্ট্রিং এবং 'toUpperCase()' একটি স্ট্রিং মেথড
```

#### মূল কথা:
- টাইপ এরর এড়াতে সবসময় সঠিক ডেটা টাইপ ব্যবহার করুন এবং অপারেশন করার আগে সেটি যাচাই করে নিন!

### Type Error: What It Is and How to Fix It

#### What is a Type Error?
A **Type Error** occurs when you try to perform an operation on a value that is not the correct data type for that operation. It happens when a mismatch in the expected data type leads to an invalid operation.

#### Common Causes of Type Errors:
1. **Invalid Operation on a Data Type**: For example, trying to call a string method on a number.
2. **Non-existent Methods**: Calling a method that doesn’t exist for the data type.
3. **Incorrect Usage of Data Structures**: Treating arrays or objects as numbers or strings, and vice versa.

#### Example of a Type Error:

```javascript
let num = 10;
num.toUpperCase();  // Type Error: 'toUpperCase()' is a string method, but 'num' is a number
```

#### How to Fix Type Errors:
1. **Use Correct Data Types**: Make sure you're performing operations on the right type of data.
2. **Check Method Availability**: Ensure that the method you are calling is available for the data type.
3. **Debug**: Use `console.log()` to check the data type before performing operations.

#### Corrected Code:

```javascript
let text = "Hello";
text.toUpperCase();  // Works fine because 'text' is a string, and 'toUpperCase()' is a string method
```

#### Key Takeaway:
- Always ensure that the data type is correct for the operation you want to perform to avoid Type Errors!
